# Results

The `Results` component is used to display results response from a search query.

```js
import { Results } from '@sajari/react-search-ui';
```

## Usage

Additional imports used in the examples:

```js
import { FieldDictionary, Pipeline, SearchProvider } from '@sajari/react-search-ui';
```

```jsx
function Example() {
  const pipeline = new Pipeline(
    {
      account: '1594153711901724220',
      collection: 'bestbuy',
    },
    'query',
  );

  return (
    <SearchProvider
      search={{
        pipeline,
        fields: new FieldDictionary({
          title: 'name',
          subtitle: (data) => data.level4 || data.level3 || data.level2 || data.level1 || data.brand,
        }),
        variables: new Variables({
          resultsPerPage: 5,
          q: 'iphone',
        }),
      }}
      searchOnLoad
    >
      <Results />
    </SearchProvider>
  );
}
```

### Product status

Currently three statuses are being supported, they are "out of stock", "sale" and "new arrival", they are hidden by default, to show them pass the `showStatus` prop,

See below list for implementation logic, the product will be:

- Out of stock if the `quantity` is 0.
- On sale if the `price` is lower than `originalPrice`.
- A new arrival if its `createdAt` is within 30 days to the current date.

```jsx
function Example() {
  const pipeline = new Pipeline(
    {
      account: '1594153711901724220',
      collection: 'bestbuy',
    },
    'query',
  );

  const fields = new FieldDictionary({
    title: 'name',
    subtitle: (data) => data.level4 || data.level3 || data.level2 || data.level1 || data.brand,
    originalPrice: (values) => {
      const price = Number(values.price);
      return Math.random() > 0.5 ? price + 100 : price;
    },
    quantity: (values) => {
      return Math.random() > 0.5 ? 0 : 1;
    },
    createdAt: (values) => {
      const newArrival = new Date();
      const old = new Date();
      old.setDate(old.getDate() - 60);
      return Math.random() > 0.5 ? newArrival : old;
    },
  });

  return (
    <SearchProvider
      search={{
        pipeline,
        fields,
      }}
      searchOnLoad
    >
      <div className="flex flex-col items-center space-y-4">
        <ViewType />
        <Results showStatus />
      </div>
    </SearchProvider>
  );
}
```

### Custom templating

You can opt-out of Sajari's default result design and go with your own, simply pass an object containing a `html` and `css` property via the `template` prop. Use `{{expression}}` syntax to embed your result's value instead, since we're using [Handlerbars](https://handlebarsjs.com/) under the hood, any default block helpers (`{{#if}}`, `{{#each}}`, ...) will also work as expected.

```jsx
function Example() {
  const pipeline = new Pipeline(
    {
      account: '1594153711901724220',
      collection: 'bestbuy',
    },
    'query',
  );

  const template = {
    html: `
<a href="{{url}}">
  <article class="relative p-3 rounded-md border border-gray-300 inline-block max-w-xs">
     <img class="mx-auto" src="{{image}}" />
     <p class="font-medium text-gray-700 mt-4">{{title}}</p>
     <p class="text-xs text-gray-400 mt-1">{{subtitle}}</p>
     <p class="text-gray-500 mt-2 text-sm">{{description}}</p>
     <div class="flex mt-8 justify-between">
       <span class="font-medium text-gray-700">\${{price}}</span>
       {{#if free_shipping}}
       <div class="flex rounded px-1 items-center text-xs text-green-500 border border-green-500 bg-green-200">
          <svg xmlns="http://www.w3.org/2000/svg" class="h-4 w-4 mr-1" viewBox="0 0 20 20" fill="currentColor">
            <path d="M8 16.5a1.5 1.5 0 11-3 0 1.5 1.5 0 013 0zM15 16.5a1.5 1.5 0 11-3 0 1.5 1.5 0 013 0z" />
            <path d="M3 4a1 1 0 00-1 1v10a1 1 0 001 1h1.05a2.5 2.5 0 014.9 0H10a1 1 0 001-1V5a1 1 0 00-1-1H3zM14 7a1 1 0 00-1 1v6.05A2.5 2.5 0 0115.95 16H17a1 1 0 001-1v-5a1 1 0 00-.293-.707l-2-2A1 1 0 0015 7h-1z" />
          </svg>
          Free shipping
       </div>
       {{else}}
       <div class="flex rounded px-1 items-center text-xs text-red-500 border border-red-500 bg-red-200">
          <svg xmlns="http://www.w3.org/2000/svg" class="h-4 w-4" viewBox="0 0 20 20" fill="currentColor">
            <path fill-rule="evenodd" d="M4.293 4.293a1 1 0 011.414 0L10 8.586l4.293-4.293a1 1 0 111.414 1.414L11.414 10l4.293 4.293a1 1 0 01-1.414 1.414L10 11.414l-4.293 4.293a1 1 0 01-1.414-1.414L8.586 10 4.293 5.707a1 1 0 010-1.414z" clip-rule="evenodd" />
          </svg>
          No free shipping
       </div>
       {{/if}}
     </div>
     <div class="new-arrival-tag text-green-500 shadow-md rounded-r border border-l-0 border-gray-200">New arrival</div>
  </article>
</a>`,
    css: `
.new-arrival-tag {
  position: absolute;
  font-size: 12px;
  line-height: 16px;
  top: 16px;
  left: 0px;
  background: white;
  padding: 2px 4px;
}`,
  };

  return (
    <SearchProvider
      search={{
        pipeline,
        fields: new FieldDictionary({
          title: 'name',
          subtitle: (data) => data.level4 || data.level3 || data.level2 || data.level1 || data.brand,
        }),
        variables: new Variables({
          resultsPerPage: 3,
          q: 'macbook',
        }),
      }}
      searchOnLoad
    >
      <Results template={template} />
    </SearchProvider>
  );
}
```

## Props

| Name                | Type                                         | Default   | Description                                                                                                                     |
| ------------------- | -------------------------------------------- | --------- | ------------------------------------------------------------------------------------------------------------------------------- |
| `appearance`        | `list` \| `grid`                             |           | The controlled value, passing this means you're ignoring the value in context.                                                  |
| `defaultAppearance` | `list` \| `grid`                             | `list`    | The uncontrolled value, will override the context value on mount.                                                               |
| `showImage`         | `boolean`                                    | `true`    | Sets to show images                                                                                                             |
| `showVariantImage`  | `boolean`                                    | `false`   | Sets to show variant images                                                                                                     |
| `showStatus`        | `boolean`                                    | `false`   | Sets to show statuses                                                                                                           |
| `imageAspectRatio`  | `number` \| `{ list: number, grid: number}`  | `1`       | The aspect ratio to force images to render to.                                                                                  |
| `imageObjectFit`    | `string` \| `{ list: string, grid: string }` | `contain` | The aspect ratio to force images to render to. Valid `objectFit` values can be found in [image props](/components/image#props). |
| `template`                       | `{html: string, css: string}`                |           | The template code use to render a single result item.                                                                           |
| `resultContainerTemplateElement` | `"div" \| "p" \| ...`                        | `"div"`   | The element used to render the result container.                                                                                |
