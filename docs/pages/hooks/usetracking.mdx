# useTracking

`useTracking` provides methods for enabling tracking features.

```js
import { useTracking } from '@sajari/react-hooks';
```

## Usage

useTracking pulls the Search.io Pipeline, as provided by the closest [SearchProvider](searchprovider), and forwards a set of utilities which are useful for various tracking use-cases. For more in-depth on how to use it, please refer to the [Tracking](/tracking) section.

## Anatomy

`useTracking` returns an object containing the following properties:

| Name                        | Type                                                                                                 | Default | Description                                                                                                                                             |
| --------------------------- | ---------------------------------------------------------------------------------------------------- | ------- | ------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `handleResultClicked`       | `(url: string) => void`                                                                              |         | The callback to attach on result when click, can be used in conjunction with GoogleAnalytics,...                                                        |
| `consumeInteractionToken`   | `(token: string, identifier: string, weight: number,data?: Record<string, string>) => Promise<void>` |         | Method to consume pos/neg token when using PosNegTracking.                                                                                              |
| `posNegLocalStorageManager` | [PosNegLocalStorageManager](/classes/posneglocalstoragemanager)                                      |         | instance of PosNegLocalStorageManager for localstorage based management of PosNeg tokens. See [Advanced tracking](/tracking/advanced) for more details. |
